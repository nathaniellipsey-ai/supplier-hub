================================================================================
                            TROUBLESHOOTING GUIDE
================================================================================

PROBLEM: "Connection refused" when opening http://127.0.0.1:5000
  SOLUTION:
    1. Make sure START_BACKEND.bat is running
    2. Make sure START_FRONTEND.bat is running
    3. Wait 2-3 seconds for both to fully start
    4. Try refreshing the browser (Ctrl+R)

PROBLEM: Frontend shows "Cannot reach API"
  SOLUTION:
    1. Verify backend is running: http://127.0.0.1:8000/health
    2. Check that port 8000 is open
    3. Look at backend terminal for errors
    4. Try restarting both servers

PROBLEM: Port already in use
  SOLUTION:
    1. Kill existing processes on that port
    2. Or change the port in the batch files
    3. Update API_BASE_URL if you change backend port

PROBLEM: No suppliers showing
  SOLUTION:
    1. Check browser console for errors (F12)
    2. Verify backend is responding to API calls
    3. Try http://127.0.0.1:8000/api/dashboard/suppliers
    4. Check backend terminal for errors

PROBLEM: "Module not found" error
  SOLUTION:
    1. The batch files auto-install dependencies
    2. Or manually: pip install -r requirements.txt
    3. Make sure you're in the right directory
