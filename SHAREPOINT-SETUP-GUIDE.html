üèóÔ∏è Setup Steps

1 Navigate to SharePoint Site

Go to: https://walmart.sharepoint.com/sites/SupplierPortal

If the site doesn't exist, create it:

  1. Go to SharePoint home
  2. Click "+ Create site"
  3. Choose "Team site"
  4. Name: Supplier Portal
  5. URL: SupplierPortal

2 Create List 1: UserProfiles

This list stores user profile information.

  1. On the SharePoint site, click "+ New" ‚Üí "List"
  2. Choose "Blank list"
  3. Name: UserProfiles
  4. Click "Create"
  5. Add the following columns:
Column Name Type Notes
Title Single line of text Store user email (default column)
FullName Single line of text User's full name
Department Single line of text User's department
LastLogin Date and Time Last login timestamp

To add a column:

  1. Click "+ Add column"
  2. Select the column type
  3. Enter the column name
  4. Click "Save"

3 Create List 2: SupplierFavorites

This list stores user favorite suppliers.

  1. Click "+ New" ‚Üí "List"
  2. Choose "Blank list"
  3. Name: SupplierFavorites
  4. Click "Create"
  5. Add the following columns:
Column Name Type Notes
Title Single line of text Store user email (default column)
SupplierId Number Supplier ID number
DateAdded Date and Time When favorite was added

4 Create List 3: SupplierNotes

This list stores user notes about suppliers.

  1. Click "+ New" ‚Üí "List"
  2. Choose "Blank list"
  3. Name: SupplierNotes
  4. Click "Create"
  5. Add the following columns:
Column Name Type Notes
Title Single line of text Store user email (default column)
SupplierId Number Supplier ID number
NoteText Multiple lines of text The actual note content (set to "Plain text")
DateCreated Date and Time When note was created
DateModified Date and Time Last modification date

5 Create List 4: SupplierInbox

This list stores user inbox messages.

  1. Click "+ New" ‚Üí "List"
  2. Choose "Blank list"
  3. Name: SupplierInbox
  4. Click "Create"
  5. Add the following columns:
Column Name Type Notes
Title Single line of text Store user email (default column)
MessageId Number Unique message ID
Vendor Single line of text Vendor name
Subject Single line of text Message subject
Message Multiple lines of text Message content (set to "Plain text")
DateReceived Date and Time When message was received
IsRead Yes/No Whether message has been read
IsStarred Yes/No Whether message is starred

6 Verify Setup

After creating all lists, verify your SharePoint site has:

  • ‚úÖ UserProfiles list with 4 columns
  • ‚úÖ SupplierFavorites list with 3 columns
  • ‚úÖ SupplierNotes list with 5 columns
  • ‚úÖ SupplierInbox list with 8 columns

üéâ Setup Complete!

Your SharePoint backend is now ready! The dashboard will automatically detect and use SharePoint when available.

What happens next:

  • When users open the dashboard, it will connect to SharePoint automatically
  • The status indicator will show "‚òÅÔ∏è Cloud Mode" if connected
  • If SharePoint is unavailable, it will show "üíæ Local Mode" and use localStorage
  • All favorites, notes, and inbox items will be synced to SharePoint

üîß Configuration (Optional)

Change SharePoint Site URL

If you want to use a different SharePoint site, edit the supplier-search-engine.html file:

const SHAREPOINT_CONFIG = { siteUrl: 'https://walmart.sharepoint.com/sites/YourSiteName', lists: { userProfiles: 'UserProfiles', favorites: 'SupplierFavorites', notes: 'SupplierNotes', inbox: 'SupplierInbox' }, enabled: true, useLocalStorageFallback: true };

Disable SharePoint (Use localStorage Only)

To temporarily disable SharePoint and use only localStorage:

const SHAREPOINT_CONFIG = { siteUrl: 'https://walmart.sharepoint.com/sites/SupplierPortal', lists: { ... }, enabled: false, // Set to false useLocalStorageFallback: true };

üêõ Troubleshooting

‚ùå Dashboard shows "üíæ Local Mode" instead of "‚òÅÔ∏è Cloud Mode"

  • Check that the SharePoint site URL is correct
  • Verify all 4 lists exist and are named correctly
  • Check browser console for errors (F12)
  • Ensure you're logged into Walmart's network/VPN
  • Check that SHAREPOINT_CONFIG.enabled is true

‚ùå CORS or authentication errors

  • Ensure you're accessing the dashboard from a Walmart domain or localhost
  • Check that you're signed into Microsoft 365 with your Walmart account
  • Try opening the SharePoint site directly first, then reload the dashboard

‚ùå Data not syncing

  • Check browser console for API errors
  • Verify column names match exactly (case-sensitive)
  • Ensure you have Edit permissions on the SharePoint lists
