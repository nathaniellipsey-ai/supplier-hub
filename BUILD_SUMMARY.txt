HOW IT WORKS
====================================================================

START
-----
1. User runs: start_server.bat (or python app.py)
2. app.py starts FastAPI server
3. suppliers.py generates 500 seeded suppliers on startup
4. Server listens on port 8000
5. Frontend served at http://localhost:8000

FRONTEND REQUEST
----------------
1. User opens http://localhost:8000
2. app.py serves dashboard_with_api.html
3. HTML loads CSS, JavaScript
4. JavaScript runs and initializes dashboard

BACKEND API REQUEST
-------------------
1. Frontend calls /api/suppliers?search=lumber
2. app.py receives request
3. API endpoint processes filters
4. Returns JSON with matching suppliers
5. Frontend renders supplier cards

LOCAL STORAGE
--------------
1. User clicks "Add to Favorites"
2. Frontend saves to localStorage (browser)
3. User's favorites persist across sessions
4. No server storage needed

====================================================================
API ENDPOINTS
====================================================================

GET /api/suppliers
  Query: skip, limit, search, category, region, verified_only, min_rating, min_ai_score
  Returns: Paginated list of suppliers

GET /api/suppliers/{id}
  Returns: Single supplier details

GET /api/stats
  Returns: Dashboard statistics (total, verified, avg rating, avg AI score)

GET /api/categories
  Returns: List of all supplier categories

GET /api/regions
  Returns: List of all supplier regions

GET /api/health
  Returns: {"status": "ok", "message": "..."}

GET /
  Returns: dashboard_with_api.html (frontend)

GET /{path}
  Returns: Static files (HTML, CSS, JS, etc.)
